GIT 基本操作

git-command.jpeg

说明：

  • workspace：工作区
  • staging area：暂存区
  • local repository：本地仓库
  • remote repository：远程仓库

一般工作流程

  1. 将项目拷贝到本地

    git clone [url]

    拉取远程仓库的新提交：

    git fetch  // 不会自动合并分支
git pull  // 拉取远程仓库上当前分支的最新提交，并合并到本地分支上
git pull [remote-shortname] [local-branch]

  2. 创建自己的工作分支

    git branch [new-branch-name]  // 创建分支
git checkout [new-branch-name]  // 切换到指定分支

    或：

    git checkout -b [new-branch-name]  // 创建并切换到新分支

  3. git commit

Gatling

Gatling 是一个功能强大的负载测试工具。它为易于使用，易于维护和高性能设计。

开箱即用，Gatling 提供了对 HTTP 协议的出色支持，这使它成为任何 HTTP 服务器负载测试的首选工具。由于核心引擎实际上是协议无关的，因此也可能实现对其他协议的支持。例如，Gatling 目前也提供 JMS 支持。

Elastic Observability(可观察性)

原文地址：Observability Guide
版本：7.16

Elastic Observability 是什么？

Observability 提供对正在环境中运行的应用程序行为的详细洞察和上下文。在 Elastic 中，我们将 observability（可观察性）视为任何你构建的并且想要监控的系统的一个属性。能够在可观察的系统中快速检测到根本原因事件并进行修复是任何分析器应该具备的最低能力。

Elastic Observability 提供一个单一的堆栈来统一你的日志，统计数据，运行时数据，应用程序跟踪，用户体验数据以及合成数据。将这些数据直接提取到 Elasticsearch，在这里你可以进一步处理和增强数据，然后在 Kibana 中进行可视化。

Last Published: 2021-11-20

1. Introduction

1.1 什么是POM？

POM 代表 “Project Object Model”，它用 XML 表示一个 maven 项目并保存在 pom.xml 文件中。Maven 存在的情况下，谈论一个项目不仅仅是指 code 的集合，而是包括 configuration files、相关开发人员和他们的角色、缺陷跟踪系统、组织和许可、项目 URL、项目 dependencies 以及所有赋予 code 生命的其他小部分。它像一个与项目有关的所有事物的一站式商店。事实上，在 Maven 世界中，一个项目甚至不需要包含任何代码，只需要一个 pom.xml 文件。
