安装 Gatling

1. 安装要求

Java 版本

Gatling 支持 64bits OpenJDK LTS (Long Term Support) 版本: 8, 11, 17。

不支持其他 JVMs，例如 JDK 12，client JVMs，32bits 系统或 OpenJ9。

支持语言

Gatling 3.7 支持使用 Java，Kotlin 和 Scala 编写测试。早期的 Gatling 版本只支持 Scala。

Gatling 版本

确保你使用了官网提供的最新版本的 Gatling。

特别是，不要使用可以在 maven Central 上找到的里程碑（M 版本），它们没有记录和发布，仅供内部或 Gatling Enterprise 客户使用。

2. 使用 Bundle 安装

可以将 Gatling 作为独立包使用。这种情况下，你只需要一个文本编辑器（可能带有 Java、Kotlin 或 Scala 语法）来编辑你的模拟，并且能够从命令行启动 Gatling。

查看 Gatling Open Source Page 获取下载链接。

Bundle 只支持 Java 和 Scala，不支持 Kotlin。要使用 Kotlin，需要 maven 或 gradle 项目。

解压下载的包，并使用 bin 目录中的脚本来启动 Gatling 和 Recorder。

Windows 用户：不要将 Gatling 放在 Programs 文件夹中，因为可能存在权限和路径问题。

运行 Gatling 至少需要 JDK8，建议使用最新版本。

有关操作系统 (OS) 的安装和调整的所有详细信息，参阅 operations。

Gatling 启动脚本和 Gatling maven 插件支持 JAVA_HOME 环境变量。根据设置，最终可能会运行与 java -version 版本不同的版本。如果遇到奇怪的错误，例如 Unsupported major.minor version 51.0 并且你希望运行 JDK8 或更高版本，你可能需要显式设置 JAVA_HOME 环境变量。

包的结构如下：

  • bin：Gatling 和 Recorder 的启动脚本。
  • conf：Gatling、Akka 和 Logback 的配置文件。
  • lib：Gatling 和二进制依赖文件。
  • user-files：
    • simulations：放置 Simulations 代码的位置。
    • resources：非源代码文件，例如 request bodies 的 feeders 文件和模板。
  • results：生成的测试结果。

3. 使用构建工具安装

Maven

Gatling 提供了一个官方的 maven 插件：gatling-maven-plugin。这个插件会帮你编译你的 Scala 代码并启动 Gatling 模拟。

详情参阅 maven 插件文档。

Gradle

Gatling 提供了一个官方的 gradle 插件：io.gatling.gradle。这个插件会帮你启动 Gatling 模拟。

详情参阅 gradle 插件文档。

SBT

Gatling 提供了一个官方的 sbt 插件：gatling-sbt。这个插件会帮你启动 Gatling 模拟。

详情参阅 sbt 插件文档。

4. 使用 IDE

你可以选择任何文本编辑器编辑你的模拟类，但如果你是开发人员，你可能希望将最喜欢的 IDE 与 Gatling 一起使用。

IntelliJ IDEA

IntelliJ IDEA 社区版默认启用 Java、Kotlin、maven 和 gradle 支持。

如果想使用 Scala 和 sbt，必须安装 Scala 插件。增加 scala compiler 的堆栈大小，以避免遭受 StackOverflowErrors 的困扰。Gatling 建议将 Xss 设置为 100M。

VS Code

设置 VS Code 的官方文档：

  • with Java
  • with Kotlin
  • with Scala

从 IDE 启动 Gatling 和 Recorder

所有 maven、gradle 和 sbt demo 都包含一些帮助类，你可以使用它们来触发一些 Gatling 任务。

右键单击 IDE 中的 Engine 类并启动 Gatling 负载测试引擎。模拟报告将写入 target 目录。

右键单击 IDE 中的 Recorder 类并启动 Recorder。模拟将在 src/test/sources 目录中生成。

快速入门

1. 简介

本节将

  • 使用 Gatling 对一个简单的位于云主机的网络服务器进行负载测试；
  • 介绍领域特定语言（DSL）的基本元素。

安装

查看 安装 Gatling 小节来选择适合你的安装方式。建议非开发人员从使用 bundle 安装开始，本教程使用 bundle 安装展示 Gatling 命令。

编码

Gatling 的默认编码方式为 UTF-8，如果想使用其他编码方式，你需要：

  • 使用 Recorder 时选择合适的编码。
  • 在 gatling.conf 文件中配置正确的编码。它会被用于编译你的 simulations，构建你的请求和响应。
  • 确保你的文本编辑器的编码方式是正确的。

2. 测试用例

本教程会过一遍 Gatling HTTP 的大部分特性。你会了解到 simulations，scenarios，feeders，recorder，loops 等等。

示例程序

Computer-Database: http://computer-database.gatling.io

场景

为了测试示例程序的性能，我们会创建场景，代表用户访问网站时实际发生的情况：

  1. 用户访问应用程序
  2. 用户搜索关键字 ‘macbook’
  3. 用户打开其中一个相关的模型
  4. 用户返回主页
  5. 用户翻页
  6. 用户创建一个新模型

3. 基本操作

使用 Recorder

为了方便的创建 scenario，我们使用 Recorder。Gatling 提供 Recorder 工具，可以记录你在网络应用的行为并且把它们导出为 Gatling scenario。

运行 bin 目录下的脚本启动 Recorder：

  • Linux/Unix: $GATLING_HOME/bin/recorder.sh
  • Windows: $GATLING_HOME/bin/recorder.bat

启动后，配置如何记录你的请求和应答：

Recorder Mode: HTTP Proxy

Simulation Information:

  • Package: computerdatabase
  • Class Name: BasicSimulation
  • Format: Java 8
  • Follow Redirects? checked
  • Infer HTML resources? checked
  • Automatic Referers? checked
  • Remove cache headers? checked

Filters: 点击 No static resources 按钮

配置好 recorder 之后，点击 Start! 按钮并且配置你的浏览器来使用 Gatling Recorder 的代理。

Recorder 和浏览器配置的详细信息，参阅 Recorder reference。
